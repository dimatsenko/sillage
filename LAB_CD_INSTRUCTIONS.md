# Інструкція з виконання лабораторної роботи: Continuous Deployment (CI/CD)

Цей документ містить покрокову інструкцію для завершення налаштування CI/CD пайплайну з розгортанням застосунку на Heroku та налаштуванням захисту гілки (для отримання додаткового балу).

## 🛠 Що вже автоматизовано в коді:

1. **Створено файл `Procfile`**: Знаходиться у корені проєкту. Цей файл вказує Heroku, що для запуску веб-сервера потрібно використовувати `gunicorn`.
2. **Оновлено `.github/workflows/ci.yml`**: Додано job `deploy` (розгортання на Heroku), яка:
   - Запускається **виключно** після успішного виконання тестів (`needs: build-and-test`).
   - Працює автоматично тільки при оновленні гілок `main` або `master`.
   - Використовує змінні середовища GitHub Secrets для безпечної авторизації.

---

## 🚀 Покрокові дії, які вам потрібно виконати

### Крок 1. Створення додатку на Heroku
1. Перейдіть на сайт [Heroku](https://heroku.com) та авторизуйтесь (або зареєструйтеся).
2. На панелі управління натисніть кнопку **New** (вгорі праворуч) → **Create new app**.
3. Введіть назву вашого додатку (наприклад, `sillage-web-app`). *Назва має бути унікальною.*
4. Оберіть регіон **Europe** та натисніть **Create app**.

### Крок 2. Отримання Heroku API Key
Щоб GitHub Action міг автоматично розгортати застосунок, йому потрібен ключ доступу.
1. На сайті Heroku натисніть на іконку свого профілю (у верхньому правому кутку) і виберіть **Account settings**.
2. Прокрутіть сторінку донизу, до розділу **API Key**.
3. Натисніть кнопку **Reveal**, щоб відкрити свій ключ, та скопіюйте його.

### Крок 3. Збереження змінних середовища (Secrets) у GitHub
Вам потрібно додати дані Heroku у ваш репозиторій на GitHub, щоб воркфлоу міг їх використати.
1. Відкрийте ваш репозиторій на [GitHub](https://github.com/).
2. Перейдіть у вкладку **Settings** (Налаштування репозиторію).
3. У лівому меню знайдіть розділ **Secrets and variables** і натисніть на **Actions**.
4. Натисніть зелену кнопку **New repository secret** і по черзі створіть 3 секрети:

| Name | Secret (Значення) |
| :--- | :--- |
| `HEROKU_API_KEY` | *(вставте скопійований API ключ з Кроку 2)* |
| `HEROKU_APP_NAME` | *(вставте назву вашого додатку Heroku, наприклад `sillage-web-app`)* |
| `HEROKU_EMAIL` | *(електронна пошта, на яку зареєстрований ваш акаунт Heroku)* |

### Крок 4. Налаштування змінних Django на Heroku
Ваш застосунок працює коректно лише тоді, коли має необхідні налаштування (Config Vars).
1. Поверніться на Heroku і перейдіть до свого додатку.
2. Відкрийте вкладку **Settings**.
3. Знайдіть розділ **Config Vars** і натисніть **Reveal Config Vars**.
4. Додайте наступні змінні (ключ -> значення):
   - `ALLOWED_HOSTS` = `*` *(або точний домен додатку: `sillage-web-app.herokuapp.com`)*
   - `DEBUG` = `0`
   - `SECRET_KEY` = *(вигадайте будь-який довгий випадковий набір символів)*

### Крок 5. Отримання додаткового балу (Налаштування Merge-to-Master)
Завдання передбачає додатковий 1 бал, якщо `merge` у гілку `master`/`main` буде можливим **тільки після** успішного проходження тестів. Для цього треба налаштувати правила захисту гілок:
1. У вашому репозиторії GitHub перейдіть у **Settings**.
2. У лівому меню оберіть пункт **Branches**.
3. Натисніть кнопку **Add branch protection rule**.
4. У полі **Branch name pattern** введіть `main` (або `master`, якщо ви використовуєте цю назву).
5. Поставте галочку біля **Require a pull request before merging**.
6. Поставте галочку біля **Require status checks to pass before merging**.
   - Під цим пунктом з'явиться поле пошуку (Search for status checks). Знайдіть там та виберіть `build-and-test` (назва нашої job з тестами). 
   - *Примітка: якщо цього пункту немає, просто збережіть зміни та зачекайте, поки pipeline хоча б раз відпрацює. Потім поверніться і виберіть його.*
7. Натисніть кнопку **Create** внизу сторінки для збереження.

### Крок 6. Запуск пайплайну
Коли все готово, вам потрібно зробити фіксацію (коміт) локальних змін та відправити їх на GitHub. Виконайте в терміналі проєкту:
```bash
git add .
git commit -m "Налаштовано пайплайн CD для Heroku та додано Procfile"
git push origin main
```

**Готово!** Після цього відкрийте вкладку **Actions** у вашому репозиторії на GitHub. Ви побачите, як ваш код автоматично тестується, створюється білд, а у разі успіху — відбувається розгортання на Heroku.
